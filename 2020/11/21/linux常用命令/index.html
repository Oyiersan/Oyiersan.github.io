五、linux 常用命令

一、系统配置

系统目录

  1. /bin 存放二进制可执行文件(ls,cat,mkdir等)，常用命令一般都在这里

  2. /etc：存放系统管理和配置文件

  3. /home： 存放所有用户文件的根目录，是用户主目录的基点，比如用户user的主目录就是/home/user，可以用~user表示

  4. /usr ： 用于存放系统应用程序 /usr/local/

  5. /var： 用于存放运行时需要改变数据的文件，也是某些大文件的溢出区，比方说各种服务的日志文件（系统启动日志等。）等；

    /etc/sysconfig/network-scripts/ifcfg-eno16777736

  • 配置网络工作

    /etc/sysconfig/network

  • 配置公共DNS服务(可选)

    /etc/resolv.conf

  • 重启网络服务

Linux的权限命令

chmod u=rwx,g=rw,o=r aaa.txt

chmod 761 aa.txt

补充一个比较常用的东西:

假如我们装了一个zookeeper，我们每次开机到要求其自动启动该怎么办？

  1. 新建一个脚本zookeeper

  2. 为新建的脚本zookeeper添加可执行权限，命令是:chmod +x zookeeper

  3. 把zookeeper这个脚本添加到开机启动项里面，命令是：chkconfig –add zookeeper

  4. 如果想看看是否添加成功，命令是：chkconfig –list

crontab 命令

https://www.runoob.com/linux/linux-comm-crontab.html

-u user 是指设定指定 user 的时程表，如果不使用 -u user 的话，就是表示设定自己的时程表。

58 23 * * * cd /root/bak;/root/bak/test_dy_dump.sh;

每天23：58执行的sh脚本

参数说明：

  • -e : 执行编辑器来设定时程表
  • -r : 删除目前的时程表
  • -l : 列出目前的时程表
  • service crond start //启动服务
  • service crond stop //关闭服务
  • service crond restart //重启服务
  • service crond reload //重新载入配置
  • service crond status //查看服务状态
