记录个人blog的搭建过程

发表于 2019-03-06 更新于 2020-08-12 分类于 Hexo

使用自己的域名，不需要购买服务器，搭建一个属于自己的网站的博客是一件很cool的事情吧！利用GitHub的GitHub Pages功能，创建一个repo，来托管自己提交的静态文件。下面记录下，搭建属于自己blog的过程。

GitHub Pages

参考官方的文档GitHub Pages已经很详细了，只是记录下自己遇到坑。

  • Repository name 以username.github.io的形式命名，且注意大小写。
  • 建立的repository不能设置为私用，只有是付费的GitHub才能设置为私有的。
  • 在custom domain 设置自己的域名保存时，会自动校验自己设置的域名是否可用，同时会在repo里面创建一个CNAME文件（CNAME文件用来做域名映射用，访问username.github.com时会映射到自己的域名上），内容为自己保存的域名。免费提供了HTTPS协议转换，如果博客中引用了http协议的css或者js可能会有影响。

域名购买

域名购买的供应商平台有很多，考虑到天朝国情🐶，选择了在国外的GoDaddy平台购买，注册的详细也不是很麻烦。

  • 登录GoDaddy配置DNS,A记录里面配置自己username.github.io的IP地址。在CNAME里面可以自定义一个自己的子域名，自己设置的blog，其他的域名服务器选择的是默认设置，有时访问慢不知道是不是跟这有关。

搭建Hexo

搭建Hexo的官方文档很详细，同时也提供了中文的查看也很方便。主题选择有很多，自己选择使用较多风格简洁的NexT，样式的调整是参考大神doublemine的。由于自己修改样式的调整是直接修改的源文件，为了整体的维护下可以在themes\next\source\css\_custom\custom.styl文件里重写样式。

  • 文章图片的引用，由于通过md格式语法只能通过url的方式引用，相对路径的引用方式会用问题。因为自己不想要使用图传的方式，选择Hexo自带的标签插件来引用资源。在主配置文件_config.yml下开启资源文件post_asset_folder: true，然后使用图片资源引用标签{% asset_img example.jpg This is an example image %}，可惜不能在visual code 上做到实时预览的形式。
